# 自定义草稿名称功能说明

## 功能概述

从本次更新开始，`create_draft` 接口支持自定义草稿名称。用户可以为创建的草稿指定一个有意义的名称，系统会自动在名称后添加时间戳以确保唯一性。

## 主要特性

### 1. 可选参数
- 参数名：`draft_name`
- 类型：`string`
- 必填：否
- 默认行为：不提供时自动生成（格式：`{时间戳}{UUID}`）

### 2. 名称格式
当提供自定义名称时，最终生成的草稿 ID 格式为：
```
{自定义名称}_{时间戳}
```

**示例：**
- 输入：`我的视频项目`
- 输出：`我的视频项目_20260630143025`

### 3. 自动清理特殊字符
为确保文件系统兼容性，系统会自动清理不安全的字符：

**会被替换的字符：**
```
\ / : * ? " < > |
以及所有控制字符 (\x00-\x1f)
```

**替换规则：**
- 所有不安全字符统一替换为下划线 `_`
- 首尾空格和点号会被去除（Windows 文件系统要求）

**清理示例：**
```
输入: "项目:2024/测试*视频?"
输出: "项目_2024_测试_视频__20260630143025"
```

### 4. 长度限制
- 自定义名称最多保留前 **50 个字符**
- 超出部分会被自动截断
- 时间戳不计入长度限制

### 5. 兜底机制
- 如果清理后名称为空，使用默认名称 `draft`
- 最终格式：`draft_20260630143025`

## 应用场景

### 1. 项目管理
```python
# 按项目名称创建草稿
draft_name = f"客户A_{project_name}"
response = create_draft(draft_name=draft_name)
```

### 2. 批量处理
```python
# 批量创建时添加序号和描述
for i, item in enumerate(items):
    draft_name = f"批次001_项目{i+1}_{item['title']}"
    response = create_draft(draft_name=draft_name)
```

### 3. 用户自定义
```python
# 直接使用用户输入的名称
user_input = request.form.get("project_name")
response = create_draft(draft_name=user_input)
```

## 注意事项

1. **唯一性保证**：即使使用相同的自定义名称，时间戳也会确保每次生成的 ID 唯一
2. **字符安全**：无需手动清理特殊字符，系统会自动处理
3. **长度控制**：建议自定义名称不超过 50 字符
4. **向后兼容**：不提供 `draft_name` 参数时，行为与之前完全一致
5. **日志记录**：系统会记录自定义名称的清理过程，便于调试

```python
def _sanitize_draft_name(name: str) -> str:
    """清理草稿名称，移除文件系统不安全的字符"""
    # 移除不安全字符
    sanitized = re.sub(r'[<>:"/\\|?*\x00-\x1f]', '_', name)
    # 去除首尾空格和点号
    sanitized = sanitized.strip(' .')
    # 兜底处理
    if not sanitized:
        return "draft"
    # 长度限制
    return sanitized[:50]
```
